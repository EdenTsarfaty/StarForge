What are you selling?

The store sells futuristic spacecraft parts, upgrades, and rare items that users can use to improve their ships or gain special advantages.

What additional pages did you add? How to operate them?
  • Cargo:
    Once per day you can “unload” your vessel's cargo. The cargo is filled with pre-determined random items, each with its own rarity factor and possible quantity.
    Items can then be sold for store credits.
  • Ship Hangar:
    Displays a visual representation of your ship with five different components. Each component's condition deteriorates daily and must be repaired.
    Repaired components can also be upgraded up to level 3.
  • Market:
    An auction house where players can post new auctions and bid on those posted by others.
    When the auction's end time has passed, an admin can close it. Closing an auction grants the winning user (if there is one) the item.
    Only admins can view expired auctions.
  • Profile:
    A page where users can view all the data they entered upon registration (excluding username) and update it.
    Password changes require verifying the existing password and following a strict password policy.

Specify all the different routes your app supports
  • Authentication routes:
    • POST /register - create a new user
    • POST /login - authenticate user
    • GET /logout - end session
    • GET /session - check logged in and admin privileges
  • Store / Cart / Checkout / Payment Routes
    • GET /products - list products for sale
    • POST /cart/add - adds an item to the cart
    • DELETE /cart/:id - removes an item from the cart corresponding with id
    • GET /cart-load - list user cart items
    • POST /pay - pays for the checked out items
  • Admin Routes
    • GET /admin/log/load - retrieves activity log
    • GET /admin/product/get - loads the details of a spcific item
    • DELETE /admin/product/delete/:id - removes an item from the store corresponding with id
    • POST /admin/product/add - adds a new item to the store
  • My Items Route
    • GET /myitems/load - list purchase history
  • Cargo Routes
    • GET /cargo/load-log - retrieves current list of items
    • POST /cargo/sell - sells a specific item
    • POST /cargo/sellAll - sells all items
    • POST /cargo/unload - randomizes and adds new items to crago
  • Ship Hangar Routes
    • GET /ship/telemetry - get current ship components status
    • POST /ship/repair - repairs an item
    • POST /ship/upgrade - upgrades an item
  • Market Routes
    • GET /market/load - list all live (or pending admin) auctions
    • POST /auction/bid - places a bid on an auction
    • POST /auction/close - closes an expired auction (admin only)
    • POST /auction/post - post a new auction on market
  • Profile Routes
    • GET /profile/load - list all profile details
    • POST /profile/edit - edits a profile attribute
Explain how did you test it

For testing I took two approaches:
For the first one, I thoroughly tested the website through all the possible API fetches and frontend behaviour upon ending the project.
This was also prominent through a large part of the development where I tested on the go.

For the second approach, I wrote a very extensive prompt for chatGPT to use node fetch to test all my APIs calls. This was mostly technical and repeating code, which was over 500 lines long, so it was perfect for AI, however it did require a lot of time and work to adjust after all.
